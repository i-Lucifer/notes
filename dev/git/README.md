### git和svn的区别

- svn中心化
- git去中心化

### rebase和merge区别

| 命令   | 区别                                               | 区别二                                              |
| ------ | -------------------------------------------------- | --------------------------------------------------- |
| merge  | 产生两条线，线上单路记录每次提交的节点             | 合并后，如果子分支删除，则主分支只记录了merge命令点 |
| rebase | 产生一条提交线，线上记录着所有提交接点，永远一条线 | 没有子分支，所有记录完整保留                        |

- 使用哪个好
  - 如果你想有个稳定的，健壮的代码记录版本，应该使用rebase
  - Rebase可以提供一套清晰的代码历史
  - rebase最大的好处，并不是消除merge，而是避免merge的交织
  - 使用rebase还是使用merge，更多的是管理风格的事情
- 现状
  - 只有极少数公司使用rebase
